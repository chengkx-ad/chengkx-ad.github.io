Mybatis—持久层框架

🌱传统java程序操作数据库

连接数据库，发送sql语句，完成增删改查。主要有以下问题存在🌱📚🐇

🌱mybatis是一个持久层（Dao层）框架

它可以在java和sql之间提供更灵活的映射方案

将java程序和sql操作剥离（解耦），将sql操作、方法配置到xml文件中

🌱mybatis-config.xml文件详解

1、properties文件

2、settings全局参数定义

3、配置别名

也可以配置包名

4、typehandlers类型处理器

5、注册配置

可以直接将整个文件夹注册，相当于该文件下下所有的文件全部注册成功

🌱xxxMapper.xml文件详解-SQL映射文件

1、parameter type

​ 传入简单类型，比如按照 id 查 Monster

​ 传入 POJO 类型，查询时需要有多个筛选条件

​ 当有多个条件时，传入的参数就是 Pojo 类型的 Java 对象,比如这里的 Monster 对象

​ 当传入的参数类是 String 时，也可以使用 ${} 来接收参数

通过name或者id查找monster

用${}$接收string

3、ResultMap结果集合

要解决的问题：如果对象属性名和表的字段名一致，那么按照正常的方式即可返回结果；但是如果对象属性名和字段名不一致，会将不一致的字段设为默认值，也就是null

配置方法 如果是在mybatis-plus中可以用注解解决属性名和字段名、表名和类名不一致的问题

🌱动态SQL语句

更复杂的查询业务

1、if标签

这个if针对的是入参age，如果符合条件执行if里的sql语句，不符合就只执行外边的sql

2、where标签

where标签会自动添加where字段，多余的AND会自动删除

3、choose/when/otherwise标签

4、forEach标签

5、trim标签（不常用）

替代where、set标签

6、set标签（重点）
